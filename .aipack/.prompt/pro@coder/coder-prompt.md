Create skill files in the doc/skill folder, each one should be a markdown file that will explain how to do something, and will be passed as context for the llm.
There'll be a second llm that will be used to generate a skill description, so you can focus on writing about the skill.

Assumptions:

- A project will use the esrc framework to declare aggregates, commands and events in a base domain folder.
- A vertical slice in a project, the slice is a feature that focus on one specific task "dashboard_of_operations", "send_email_to_user", "create_operation".
- The developer/llm responsible for implementing the slice will always have the entire "bounded context"(aggregates, commands, events) as a context.
- you cannot import code from other vertical slices inside another.
- the domain will also expose a const for bounded_context and domain_name.

```rs
# <bounded_context_name>/domain/mod.rs
const BOUNDED_CONTEXT_NAME: &str = <bounded_context_name>;
# <bounded_context_name>/domain/<domain_name>/mod.rs
const DOMAIN_NAME: &str = <domain_name>;
```

Create how many skills you think are necessary, these skills should describe how to create the basic building blocks of a slice:

- Create Automations and Read models with the "event_modeling" module of the esrc.
- Execute commands with the "command_service" from esrc.
- always declare the feature_name as a const in the root of the slice. (const FEATURE_NAME: &str = "criar_batch")
  - this constant will be used for instrumentation and other configuration of the slice.
- Read models will always have the public interface defined:

```rs
# <slice_folder>/generated.rs
/// Code generated by v8digital.em-rust-code-gen.
use serde::{Deserialize, Serialize};
use uuid::Uuid;
#[derive(Debug, Clone, Default, PartialEq, Serialize, Deserialize)]
pub struct Rede {
    pub vendedor_id: Uuid,
    pub responsavel_id: Uuid,
    pub tipo: String,
}
#[derive(Debug, Serialize, Deserialize, Clone)]
pub struct RedeQuery {
    pub vendedor_id: Option<Uuid>,
    pub responsavel_id: Option<Uuid>,
}
```
- Queries are public interfaces for some slices, some are generated, if a read model is required to expose a custom more complex query, it declares it in the mod.rs where it will be exposed together with the generated ones.
```rs
# <slice_folder>/mod.rs
/// Code generated by v8digital.em-rust-code-gen.
#[derive(Debug, Serialize, Deserialize, Clone)]
pub enum Queries {
    Default { query: RedeQuery },

    ByMaster { query: RedeByMasterQuery },
}

#[derive(Debug, Default, Serialize, Deserialize, Clone)]
pub struct RedeByMasterQuery {
    pub master_id: Uuid,
}
```
